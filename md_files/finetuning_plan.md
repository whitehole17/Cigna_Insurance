# 파인튜닝 계획서

## 목적 및 범위

이 프로젝트는 **LangGraph 기반 RAG 파이프라인**을 사용하므로, 일반적인 보험 Q&A 답변은 파인튜닝 대상이 아닙니다. LangGraph가 부족한 정보를 사용자에게 재요청하고, RAG가 사실 정보를 제공하는 구조이기 때문입니다.

파인튜닝은 두 가지 **행동 제어** 목적으로만 사용합니다.

| 목적 | 설명 |
|------|------|
| ① 면책 고지 자동 삽입 | 사용자가 압박하거나 확정을 요구해도 절대 생략하지 않도록 |
| ② 보험 추천 거절 | 어떤 방식으로 질문해도 추천 행위로 이어지지 않도록 |

---

## 데이터 규모 결정

"새로운 지식 주입"이 아니라 "행동 패턴 교정"이 목적이므로, 일반 도메인 파인튜닝(800~1,000개)보다 적은 양으로 충분합니다.

| 목적 | 시드(직접 작성) | 증강 후 목표 |
|------|--------------|------------|
| 면책 고지 고정 | 20개 | 100~150개 |
| 추천 거절 | 20개 | 80~100개 |
| **합계** | **40개** | **약 200~250개** |

Qwen2.5-7B + QLoRA + unsloth 기준, Colab 무료 T4에서 200개 학습 시 약 30분 소요.

---

## 데이터 구성 전략

### 황금 비율

- **공통 형식 학습 (60%)**: 모든 보험사에 공통 적용되는 면책 고지 및 정보 제공형 답변
- **보험사별 특화 (30%)**: 각 보험사 고유 용어 반영 (Cigna의 Cost Share, TRICARE의 cost-share, Allianz의 Co-payment 등)
- **거절·방어 (10%)**: 추천 거절, 확정 강요 거절

### 면책 고지 원칙

핵심 3요소는 100% 고정하되, 문장 표현에는 약간의 variation을 허용합니다 (과적합 방지).

- ⚠️ 마커 포함
- "공개 문서 기준" 명시
- "직접 확인" 권고 (보험사명 포함)

> ❌ 면책 문구를 완전히 동일하게 고정하면 오히려 과적합(overfitting) 위험이 생깁니다.
> ✅ 핵심 요소는 고정하되, 문장 시작은 다양하게 구성합니다.

### 답변 시작 다양화 예시

- A타입: "문의하신 내용은 [출처]에 따르면 다음과 같습니다."
- B타입: "[출처]를 확인한 결과, 해당 보장은..."
- C타입: "확인해 주신 플랜의 상세 조건은 [출처] 기준..."

## JSONL 포맷

Qwen2.5 / Gemma2 chat 모델에 맞는 **messages 배열(ChatML)** 형식을 사용합니다.
Alpaca 스타일(`instruction / input / output`)은 사용하지 않습니다.

```jsonl
{"messages": [
  {"role": "system", "content": "당신은 재한 외국인을 위한 국제 건강보험 심리상담 Coverage 안내 어시스턴트입니다. ..."},
  {"role": "user", "content": "질문"},
  {"role": "assistant", "content": "답변"}
]}
```

## 데이터 증강 계획

1. 핵심 골드 샘플 각 20개씩 직접 작성 (총 40개)
2. GPT-4로 증강

```
[면책 고지 증강 프롬프트]
아래 면책 고지 학습 데이터 20개를 참고해서, 동일한 규칙
(⚠️ 마커 포함, "공개 문서 기준" 명시, "직접 확인" 권고)을 유지하면서
질문 표현과 답변 시작 문장을 다양하게 변형한 항목 130개를 추가로 만들어줘.
보험사는 Cigna, TRICARE, Allianz, Bupa를 균등하게 포함해줘.

[추천 거절 증강 프롬프트]
아래 추천 거절 데이터 20개를 참고해서, 사용자가 다양한 방식으로
특정 플랜을 추천받으려 시도하는 질문 80개와 그에 대한 거절 답변을 추가로 만들어줘.
우회·비공식 요청, 전문가 의견 요청, 가성비 비교 등 다양한 유형을 포함해줘.
```

3. 검수: 면책 고지가 누락된 항목, 추천 행위가 포함된 항목 즉시 제거

## 학습 환경 및 코드 스니펫

| 환경 | 모델 | VRAM | 학습 시간 | 비용 |
|------|------|------|---------|------|
| Colab 무료 T4 | Qwen2.5-7B + QLoRA | ~10GB | ~30분 | 무료 |
| Colab Pro / RunPod | Gemma2-9B + QLoRA | ~14GB | ~25분 | ~$0.30 |

**권장:** 자원 제약 시 Colab 무료 T4 + Qwen2.5-7B

```python
from unsloth import FastLanguageModel

model, tokenizer = FastLanguageModel.from_pretrained(
    model_name="unsloth/Qwen2.5-7B-Instruct",
    max_seq_length=2048,
    load_in_4bit=True,  # QLoRA: 4bit 양자화로 VRAM 절감
)

model = FastLanguageModel.get_peft_model(
    model,
    r=16,
    lora_alpha=16,
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj"],
)
```

## 파인튜닝 전/후 기대 비교

| 질문 유형 | 파인튜닝 전 | 파인튜닝 후 |
|----------|----------|----------|
| Coverage 질문 | "20세션 커버됩니다." | "[문서 p.26 기준] 20세션. ⚠️ 실제 적용은 보험사 확인 필요." |
| "확실히 말해줘" 압박 | 면책 고지 생략하고 단정 | 면책 고지 유지하며 이유 설명 |
| 추천 요청 | "A플랜이 더 좋아 보입니다." | "추천은 제공하지 않습니다. 조건 비교를 도와드릴까요?" |
| 우회 추천 요청 | 추천에 응함 | 우회 패턴 감지 후 거절 및 비교 안내로 전환 |
